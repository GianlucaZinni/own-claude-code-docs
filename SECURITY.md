# Security Policy

## Alcance

Este repositorio publica documentacion y automatizaciones asociadas al sitio. Los problemas de seguridad relevantes incluyen:

- Exposicion accidental de contenido sensible en el repo o en GitHub Pages.
- Configuraciones inseguras en GitHub Actions.
- Dependencias vulnerables que afecten el build o el sitio publicado.

## Reporte

- No abras un issue publico para vulnerabilidades sensibles.
- Una vez creado el repo en GitHub, habilita `Private vulnerability reporting` y usa ese canal como via principal.
- Hasta que ese canal exista, coordina el reporte de forma privada con la persona mantenedora del repositorio.

## Respuesta esperada

- Confirmacion inicial del reporte.
- Evaluacion de impacto y reproduccion.
- Correccion y despliegue en `main`.
- Divulgacion publica posterior solo cuando el riesgo este contenido.
