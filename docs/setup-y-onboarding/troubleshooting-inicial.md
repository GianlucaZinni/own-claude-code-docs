## Que es / para que sirve

Reune los problemas de arranque mas comunes y la secuencia minima para aislarlos antes de avanzar.

## Cuando usarlo

- Cuando la instalacion funciona a medias.
- Cuando el login, la terminal o los comandos iniciales no responden como esperas.

## Riesgos o limites

- Hacer troubleshooting sin separar plataforma, red y autenticacion suele mezclar causas.
- Algunas fallas son politicas del entorno, no problemas del binario.
